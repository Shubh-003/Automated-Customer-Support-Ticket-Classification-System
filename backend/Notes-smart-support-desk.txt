JWT Advance:
Login →
   Access Token (15 min)
   Refresh Token (7 days)
Access expires →
   Client calls /refresh →
   New Access Token generated
Logout →
   Refresh token revoked

# TARGET DESIGN for JWT:
 Access Token:
   15 minutes
   Used on every API call
 Refresh Token:
  Stored in DB
  7 days expiry
  Used only to generate new access token

  Token Rotation (Advanced Security)
    When refreshing:
    Instead of just issuing new access token:
    Revoke old refresh token
    Create new refresh token
  This prevents replay attacks.

Blacklist Access Tokens (Optional but Elite Level):

Access Token → short-lived → used for APIs
Refresh Token → long-lived → used to get new access
When refresh token dies:
Session is over.

ROLE-BASED ACCESS CONTROL (RBAC) Autherization:
@EnableMethodSecurity allow to use @PreAuthorize
@PreAuthorize("hasAnyRole('USER','AGENT','ADMIN')") -> createTicket
@PreAuthorize("hasAnyRole('ADMIN','AGENT')") -> viewAllTicket
@PreAuthorize("hasRole('ADMIN')") -> deleteTicket
RoleHierarchy Bean in securityConfig
