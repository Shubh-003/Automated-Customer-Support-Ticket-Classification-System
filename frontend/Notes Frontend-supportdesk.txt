## How will USER vs ADMIN/AGENT UI be different?
We will read the role from JWT login response.
Your backend already returns role inside the token (or user info).
Example login response:
{
  "accessToken": "...",
  "refreshToken": "...",
  "role": "ADMIN"
}

We store role in:
AuthContext

Example Auth State
{
  user: "shubh",
  role: "ADMIN",
  token: "..."
